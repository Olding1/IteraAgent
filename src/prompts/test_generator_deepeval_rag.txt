## 要求

1. **事实性问题**: 问题应该是明确的事实性问题,有唯一的正确答案
2. **可验证性**: 答案必须能在文档中找到,不能是推测或推理
3. **覆盖性**: 问题应该覆盖文档的不同部分和不同主题
4. **自然性**: 问题应该符合用户真实的提问方式,不要过于生硬
5. **多样性**: 问题类型应该多样化 (What, How, When, Where, Why)

## 输出格式 (⚠️ 严格要求)

**你必须输出一个有效的 JSON 数组**,每个问答对包含:
- `question`: 问题内容
- `expected_answer`: 预期答案 (从文档中提取的准确答案)
- `source_section`: 答案来源的文档章节或段落 (可选)

### ✅ 正确示例:
```json
[
  {{
    "question": "IteraAgent 的核心目标是什么?",
    "expected_answer": "IteraAgent 的核心目标是打造一个智能体构建工厂,通过自然语言需求自动生成可执行的 LangGraph Agent。",
    "source_section": "项目愿景"
  }},
  {{
    "question": "Phase 4 主要实现什么功能?",
    "expected_answer": "Phase 4 主要实现闭环与进化功能,包括测试生成、执行、评估和自动修复。",
    "source_section": "Phase 4 - 闭环与进化"
  }}
]
```

### ❌ 常见错误 (请避免):
```json
// 错误 1: 缺少花括号
[
    "question": "...",  // ❌ 应该是 {{"question": "...",
    "expected_answer": "..."
]

// 错误 2: 对象间缺少花括号
[
  {{
    "question": "...",
    "expected_answer": "..."
  }},
  "question": "...",  // ❌ 应该是 {{"question": "...",
  "expected_answer": "..."
]
```

### 📋 格式检查清单:
1. ✅ 使用方括号 `[]` 包裹整个数组
2. ✅ 使用花括号 `{{}}` 包裹每个问答对象
3. ✅ 对象之间用逗号 `,` 分隔
4. ✅ 所有字符串用双引号 `"` 包裹
5. ✅ 不要添加注释或额外文字
